# Multispectral Image Preprocessing Pipeline

## Project Overview
This project establishes a **white-box pipeline** for preprocessing multispectral imagery derived from Pix4D photogrammetry results. The primary goal is to convert raw reflectance maps into analysis-ready orthomosaics for the calculation of vegetation indices (such as NDVI, NDRE, and MSAVI).

Unlike proprietary "black box" solutions, this pipeline aims to provide full transparency and control over radiometric calibration, band alignment, and index calculation steps.

## Prerequisites
To run the current workflow and future automated scripts, the following dependencies are required:

*   **Pix4D Mapper** (for initial photogrammetry and reflectance map generation)
*   **Python 3.9+**
*   **GDAL / Rasterio** (for geospatial data manipulation)
*   **NumPy** (for array-based band math)
*   **Matplotlib** (for visualization)

## Installation

Following the steps provided from [micasence](https://github.com/micasense/imageprocessing?tab=readme-ov-file).

### Windows setup
When installing on Windows we rely on the [Anaconda](https://www.anaconda.com/download) Python environment to do most of the heavy lifting for us.

- Install [Anaconda](https://www.anaconda.com/download) for your system by downloading the Python 3.6 version
    - When installing Anaconda, choose "install for only me" instead of "install for all users," as this simplifies installation of other packages

- Download the [exiftool windows package](https://exiftool.org/) and unzip it to a permanent location such as `c:\exiftool\`. Now we need to tell the Python code where to find exiftool (so we don't have to set it up in every script we write), and we do that by adding the path to exiftool as an environment variable.
    - Create an environment variable called `exiftoolpath` with a value of the full path to exiftool. For example, `c:\exiftool\exiftool.exe`
    - To do this on Windows 10, press Start button or the Windows key, then type `Path` and click `Edit Environment Variables for Your Account`
    - Click `New`
    - In Variable Name type `exiftoolpath`
    - In Variable Value type `c:\exiftool\exiftool.exe`

- Open an Anaconda console from the start menu as an administrator by clicking `Start->Anaconda`, right-click `Anaconda Console`, choose `Run as Administrator`. Execute the following commands in the anaconda console:
    - `cd` to the directory you git cloned this repository to
    - `conda env create -f micasense_conda_env.yml`
    - This will configure an anaconda environment with all of the required tools and libraries This will take a while depending on your computer and internet speeds (5-10 minutes is not uncommon)
    - When it's done, run `activate micasense` to activate the environment configured
    - Each time you run start a new anaconda prompt, you'll need to run `activate micasense`

## Current Workflow
*Based on "Creating Orthomosaic Image from Pix4D results"*

Currently, the process involves the following steps to generate the final orthomosaic from Pix4D outputs:

### 1. Pix4dMapper
Import all the imergery to `Pix4DMapper` and use the `Ag multispectral template`. Run with the default setting and export the GeoTiff and Orthomosaic. 

### 2. Input Data Preparation
Locate the folder generated by Pix4D. Ensure the following individual band files (GeoTIFF) are present:
*   Blue
*   Green
*   Red
*   Red Edge
*   Near-Infrared (NIR)

### 3. Mosaic Generation
1.  Import individual band tiles into the processing environment.
2.  Merge tiles for each spectral band to create a seamless single-band orthomosaic.
3.  Ensure coordinate reference systems (CRS) match the project requirements (e.g., EPSG:4326 or UTM zones).

### 4. Band Alignment & Stacking
1.  Align the single-band orthomosaics to ensure pixel-perfect registration between bands (critical for accurate index calculation).
2.  Stack bands into a single multi-layer GeoTIFF or keep as aligned separate files.

### 5. Vegetation Index Calculation
Apply band math formulas to generate indices.

**Normalized Difference Vegetation Index (NDVI):**
$$NDVI = \frac{(NIR - Red)}{(NIR + Red)}$$

**Normalized Difference Red Edge (NDRE):**
$$NDRE = \frac{(NIR - RedEdge)}{(NIR + RedEdge)}$$

### 6. Output
The final outputs are GeoTIFF files containing the calculated indices, clipped to the region of interest (ROI).

## Roadmap: Automatic Pipeline Development
The following tasks are planned to transition this workflow from a manual process to a fully automated Python-based pipeline.

### Phase 1: Core Automation Scripting
- [ ] **CLI Development**: Create a `main.py` with `argparse` to accept input directories and output paths as command-line arguments.
- [ ] **Automated Band Detection**: Implement regex logic to automatically identify and sort Pix4D output files (e.g., `*_nir.tif`, `*_red.tif`) regardless of naming variations.
- [ ] **Batch Mosaicking**: Replace manual merging with `gdal_merge.py` or `rasterio.merge` subprocesses.

### Phase 2: Image Processing & Quality
- [ ] **Dynamic Alignment**: Implement feature matching (SIFT/ORB) to auto-correct minor misalignments between spectral bands before calculation.
- [ ] **Reflectance Normalization**: Add a module to check metadata for radiometric calibration factors and apply them if Pix4D has not fully corrected the data.
- [ ] **ROI Clipping**: Allow the user to pass a Shapefile/GeoJSON to automatically clip the resulting orthomosaic to the field boundary.

### Phase 3: DevOps & Scalability
- [ ] **Dockerization**: Create a `Dockerfile` to containerize GDAL and Python dependencies, ensuring reproducibility across different machines.
- [ ] **Watchdog Service**: Create a background service that watches a specific folder; when new Pix4D results are dropped, the pipeline triggers automatically.
- [ ] **Logging & Reporting**: Generate a PDF report after processing that shows histograms of the calculated indices and flags any potential data quality issues.
